💡 Le concept
📋

Collecte automatique

Un script Python collecte chaque nuit l'état complet de l'infra : logs SSH, services systemd, disque, mémoire, load average et jobs Slurm.

🤖

Analyse par IA

Les données brutes sont envoyées à Groq (llama-3.3-70b) qui les analyse et génère un résumé en français, classé par priorité.

📄

Rapport Markdown

Le résultat est sauvegardé dans /var/log/opslm-report/ avec la date du jour.

🔒

100% self-hosted

Tourne sur la VM privée. Seuls les logs anonymisés sont envoyés à Groq. Aucune donnée sensible ne quitte l'infra.

⚙️ Installation
1

Prérequis

Python 3, la lib groq, et une clé API Groq gratuite sur console.groq.com. OpsLM doit être installé dans /opt/opslm/.

2

Déploiement du cron

Le script install_report.sh installe automatiquement le cron job à 06h00 et crée le dossier /var/log/opslm-report/.

3

Premier rapport

Lance le script manuellement pour vérifier. Le fichier YYYY-MM-DD.md est créé immédiatement dans le dossier de logs.

bash — 3 commandes
# 1. Copier le script cp generate_report.py /opt/opslm/generate_report.py # 2. Installer le cron (06h00 chaque matin) chmod +x install_report.sh && ./install_report.sh # 3. Tester immédiatement GROQ_API_KEY=$(systemctl show opslm | grep GROQ_API_KEY | cut -d= -f3) \ /opt/opslm-env/bin/python /opt/opslm/generate_report.py # Lire le rapport généré cat /var/log/opslm-report/$(date +%Y-%m-%d).md
